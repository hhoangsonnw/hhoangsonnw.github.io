# Future Maintenance Notes

This repository is intended to run as the personal GitHub Pages site at:

```text
https://hhoangsonnw.github.io
```

## Daily Content Workflow

Add or edit writeups only under `content/`. The site does not publish random top-level folders.

After any content change, run:

```bash
npm run content:sync
npm run build
```

Then verify the generated manifest:

```bash
node -e "const m=require('./public/content-manifest.json'); for (const p of m.posts) console.log([p.title,p.url,p.difficulty,p.sourcePath].join(' | '))"
```

## Publish To GitHub Pages

The repository should be pushed to:

```text
https://github.com/hhoangsonnw/hhoangsonnw.github.io
```

GitHub Actions builds and deploys the site from `main` using:

```text
.github/workflows/deploy.yml
```

Before pushing, use:

```bash
npm run build
```

Then commit and push:

```bash
git add .
git commit -m "Update website"
git push origin main
```

On GitHub, make sure Pages is configured to deploy from GitHub Actions.

## Add A New HTB Challenge

```bash
npm run content:add -- --branch HTB/Challenges --folder --title "Challenge Name" --difficulty medium --category Challenges --tags htb,challenge,forensics
```

Replace the generated `WRITEUP.md` with the real writeup, put screenshots in the same folder, then run:

```bash
npm run content:sync
```

## Image And Asset Hygiene

Keep screenshots beside the writeup that uses them:

```text
content/HTB/Challenges/example/
├── WRITEUP.md
├── image.png
└── image-1.png
```

Use relative links in Markdown:

```markdown
![Evidence](image.png)
```

Before publishing a large batch, remove temporary files:

```bash
find content -name '.DS_Store' -delete
find content -name '*tmp.html' -delete
```

Do not commit `node_modules/` or `dist/`. They are regenerated by install/build commands.

## Troubleshooting

If a post does not appear, run:

```bash
npm run content:sync
```

If the site shows old placeholder text, check the source and generated content:

```bash
sed -n '1,40p' content/path/to/WRITEUP.md
sed -n '1,40p' public/content/path/to/WRITEUP.md
```

If routes work locally but not on GitHub Pages, confirm:

- `public/.nojekyll` exists.
- `dist/404.html` is created by `npm run build`.
- GitHub Pages is deploying from GitHub Actions.
- The workflow completed successfully on the `main` branch.

## Periodic Updates

Monthly or before a large publishing session:

```bash
npm outdated
npm audit
npm run build
```

When updating dependencies, prefer small batches and verify the site after each update:

```bash
npm update
npm run build
```

If screenshots start making the repository too large, move heavy images to external object storage and link them from Markdown using absolute URLs.
